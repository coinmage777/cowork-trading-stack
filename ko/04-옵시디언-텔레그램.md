# 04. 옵시디언 + 텔레그램

지식이 쌓여야 알파가 되고, 알림이 와야 시스템이 굴러간다. 옵시디언과 텔레그램은 작업의 두 축이다.

## 옵시디언 — 리서치 누적 시스템

옵시디언 볼트는 노트 145개 이상, wikilink 670개 이상으로 구성돼 있다. 거기에 30+ 프로젝트 리서치, 90+ 에어드랍 추적, 20+ 블로그 가이드, 매매 일지가 다 들어가 있다.

### 왜 옵시디언인가

- **로컬 파일** — 클라우드 의존 없음. .md 파일이라 git / rclone / 이메일 어디로든 옮김
- **wikilink** — `[[BTC]]` 처럼 쓰면 다른 노트와 자동 연결. 그래프 뷰로 시각화
- **플러그인 강력함** — Dataview, Templater, Calendar, Kanban, Tasks
- **마크다운** — 다른 데로 export 쉬움 (블로그 / GitHub / Notion)

### 볼트 구조

쓰는 최상위 폴더:

```
Obsidian Vault/
├── Airdrop/               # 에어드랍 추적 + Kanban + ROI
├── Content/               # 블로그 / 유튜브 / 텔레그램 콘텐츠
├── Dashboard/             # 대시보드 (홈, 메트릭, 링크)
├── Inbox/                 # 수집함 (정리 안 된 raw 노트)
├── Memory/                # AI 메모리 (Topics, Daily, Archive)
├── Projects/              # 30+ 프로젝트 리서치
├── Templates/             # 12개 템플릿 (매매일지, 주간리뷰 등)
├── Trading/               # 매매 (daily/, onchain/, 전략 가이드)
├── assets/                # 이미지
├── docs/                  # 시스템 문서 (memory, research)
└── scripts/               # 자동화 스크립트
```

### 핵심 노트 패턴

- **Dashboard/Home.md** — 모든 곳의 진입점. Dataview 쿼리로 최근 노트, 진행 중 프로젝트, 에어드랍 데드라인 자동 표시
- **Projects/{ProjectName}.md** — 한 프로젝트당 한 파일. 메커니즘 / 토크노믹스 / 리서치 / 액션 아이템
- **Trading/daily/YYYY-MM-DD.md** — 매일 매매 일지. 템플릿으로 자동 생성
- **Memory/MEMORY.md** — AI 에이전트용 메모리 인덱스 (위에서 다룬 MemKraft와 연동)

### 플러그인 셋

쓰는 8개:

1. **Dataview** — 노트를 DB처럼 쿼리. 대시보드의 핵심
2. **Templater** — 매매 일지 / 프로젝트 노트 자동 생성
3. **Calendar** — 일자별 노트 캘린더 뷰
4. **Kanban** — 에어드랍 진행 상태 보드
5. **Advanced Tables** — 마크다운 테이블 편집기
6. **Periodic Notes** — 일/주/월 노트
7. **Tasks** — `- [ ]` 체크박스를 글로벌 쿼리
8. **Tag Wrangler** — 태그 정리

### Dataview 쿼리 예시

진행 중 프로젝트 자동 목록:

````markdown
```dataview
TABLE status, last_review, tier
FROM "Projects"
WHERE status = "active"
SORT last_review DESC
```
````

이번 주 매매 PnL 합계:

````markdown
```dataview
TABLE sum(rows.pnl) as "PnL"
FROM "Trading/daily"
WHERE date >= date(today) - dur(7 days)
GROUP BY week
```
````

### 그래프 뷰

옵시디언의 진짜 무기. 노트 간 wikilink로 자동으로 그래프가 만들어진다. 볼트는 47개 핵심 노트 중심으로 클러스터가 잡혀 있어서, "이 거래소 관련 노트들" / "이 전략 관련 노트들"이 자동으로 보인다.

## 텔레그램 — 알림 + 원격 제어

텔레그램은 운영 봇의 인터페이스 절반이다. 알림, 명령, PnL 보고, 긴급 정지 — 다.

### 봇 만들기

1. 텔레그램에서 [@BotFather](https://t.me/BotFather) 검색
2. `/newbot` → 이름 입력 → 토큰 받음
3. 운영 봇과 대화 시작 → `/start`
4. `https://api.telegram.org/bot<TOKEN>/getUpdates` 접속 → `chat.id` 확인

### 환경변수

```bash
TELEGRAM_BOT_TOKEN=<your_telegram_bot_token>
TELEGRAM_CHAT_ID=<your_telegram_chat_id>
```

### 알림 코드 — 미니멀 버전

```python
import os
import aiohttp

async def notify(text: str):
    token = os.getenv("TELEGRAM_BOT_TOKEN")
    chat_id = os.getenv("TELEGRAM_CHAT_ID")
    if not (token and chat_id):
        return
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    async with aiohttp.ClientSession() as s:
        async with s.post(url, json={"chat_id": chat_id, "text": text}) as r:
            await r.text()

# 사용
await notify("BOT START")
await notify(f"Position opened: BTC short, PnL est +0.5%")
```

### Dedup / 쓰로틀링

알림 폭탄 방지가 중요하다. 패턴:

```python
class Notifier:
    def __init__(self, throttle_minutes=10):
        self._last_sent = {}
        self._throttle = throttle_minutes * 60

    async def send(self, key: str, text: str):
        now = time.time()
        if key in self._last_sent and now - self._last_sent[key] < self._throttle:
            return
        self._last_sent[key] = now
        await notify(text)
```

같은 종류의 알림이 10분 안에 두 번 가지 않게 한다.

### 알림 우선순위 정의

운영 봇이 보내는 알림 종류:

| 우선순위 | 케이스 | 빈도 |
|---------|--------|------|
| **CRITICAL** | 일일 -X% 손실, 거래소 잔고 0, BNB 부족 (가스), 봇 크래시 | 즉시 |
| **WARNING** | 거래소 비활성화, 잔고 급락, WS 끊김 | 즉시 |
| **INFO** | 봇 시작 / 종료, 일일 PnL 요약 | 1일 1회 |
| **DEBUG** | 모든 진입 / 청산 | 텔레그램 X (로그만) |

### 원격 제어 — Telegram Commander

별도 봇으로 명령을 받는 패턴. 운영해온 commander 명령어:

| 명령 | 동작 |
|------|------|
| `/status` | 봇 상태 / 포지션 / PnL 요약 |
| `/pnl` | 일일 PnL 보고 (daily_report.py 호출) |
| `/balance` | 거래소별 잔고 |
| `/positions` | 활성 포지션 목록 |
| `/restart` | 트리거 파일 생성 → 봇이 graceful restart |
| `/reload` | 트리거 파일 → config 핫 리로드 |
| `/close` | 포지션 청산 + 봇 종료 (`/yes` 2단계 확인) |
| `/kill <ex>` | 특정 거래소 비활성화 |
| `/revive <ex>` | 비활성화된 거래소 복구 |
| `/bnb` | BNB 가스 잔고 (Polymarket용) |

### 화이트리스트 보안

commander 봇은 반드시 chat_id 화이트리스트로 운영자만 명령 가능하게 해야 한다:

```python
ALLOWED_CHAT_IDS = [int(x) for x in os.getenv("TELEGRAM_ALLOWED_CHATS", "").split(",")]

async def handle(update):
    if update.message.chat.id not in ALLOWED_CHAT_IDS:
        return  # 무시
    # ...
```

이거 빠지면 누군가 운영 봇 토큰 알아내서 `/close` 날리면 끝이다.

### 트리거 파일 패턴 (Windows 친화)

Windows는 SIGHUP / SIGTERM이 외부에서 안 통한다. 그래서 파일 트리거 방식을 쓴다:

```python
# trigger_watcher.py
TRIGGER_DIR = Path("triggers")

async def watch():
    while True:
        for trigger_file in TRIGGER_DIR.glob("*.trigger"):
            kind = trigger_file.stem
            if kind == "restart":
                await graceful_restart()
            elif kind == "reload":
                await hot_reload_config()
            elif kind == "close":
                await close_all_and_exit()
            trigger_file.unlink()  # 처리 후 삭제
        await asyncio.sleep(2)
```

commander 봇은 `/restart` 받으면 `triggers/restart.trigger` 파일 만든다. 봇이 2초 안에 감지 → 처리 → 파일 삭제.

이 패턴이 OS 독립적이고, 모바일에서 텔레그램으로 봇 제어하기 깔끔하다.
