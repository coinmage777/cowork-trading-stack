# 15. 용어 정리

가이드 전체에 나오는 용어를 알파벳 순으로 정리합니다. 처음 보는 분을 위한 빠른 참조용입니다.

## A

**Adverse Selection (역선택)** — 시장에서 주문이 체결되는 경우는 상대방이 정보 우위를 가진 경우입니다. 봇 매매 시 체결 자체가 마이너스 시그널이 될 수 있습니다.

**Agent Wallet** — 메인 지갑이 트레이딩 권한을 위임한 별도 키입니다. [Hyperliquid](https://miracletrade.com/?ref=coinmage) 같은 DEX에서 메인 지갑 노출 없이 봇 운영이 가능합니다.

**Aiohttp** — Python 비동기 HTTP 라이브러리입니다. 봇이 거래소 API에 비동기 호출할 때 표준입니다.

**API Key / Secret** — 거래소 API 인증용입니다. 트레이딩 권한만 주고 출금 권한은 절대 부여하지 않습니다.

**APR / APY** — Annual Percentage Rate / Yield. 펀딩 차익 등 전략의 연환산 수익률입니다.

**Arbitrage (차익거래)** — 같은 자산이 다른 시장에서 다른 가격에 거래될 때, 양쪽 동시 진입으로 차익을 실현합니다.

**ATR (Average True Range)** — 변동성 지표입니다. 동적 사이징 / 손절 폭 결정에 사용합니다.

## B

**Backtest** — 과거 데이터에 전략을 시뮬레이션해서 성과를 측정합니다. 라이브 투입 전 필수입니다.

**Basis (베이시스)** — 현물 가격과 선물 / Perp 가격의 차이입니다. 베이시스 차익은 carry trade의 한 형태입니다.

**BBO (Best Bid/Offer)** — 오더북에서 매수 최고 / 매도 최저가입니다. 지정가 진입 시 기준점입니다.

**Bollinger Bands** — 가격의 표준편차 기반 밴드입니다. 돌파 / 회귀 시그널 보조에 사용합니다.

**Builder Code** — Hyperliquid의 라우팅 메커니즘입니다. 프론트엔드(Miracle, DreamCash 등)별 코드로 수수료를 분배합니다.

## C

**ccxt** — 100+ 거래소 통합 Python / JS 라이브러리입니다. CEX 봇의 표준입니다.

**CEX (Centralized Exchange)** — 바이낸스, OKX, Bybit 등 중앙거래소입니다.

**Circuit Breaker** — 일일 -X% 손실 시 자동 정지합니다. 무한 손실을 방지합니다.

**Clearinghouse State** — Hyperliquid 등 DEX의 사용자 잔고 / 포지션 상태입니다.

**CLOB (Central Limit Order Book)** — 중앙 오더북 매칭입니다. Polymarket이 이 구조입니다.

**Cloid** — Client Order ID. 거래소 주문 식별자입니다. Miracle 같은 프론트엔드는 cloid prefix가 필수입니다.

**Correlation (상관관계)** — 두 자산 가격의 동조성입니다. 페어 트레이딩의 레짐 필터에 사용합니다.

**Cross-Venue** — 거래소 간을 의미합니다. Cross-venue arb = 거래소 간 가격 차이 차익입니다.

## D

**DCA (Dollar Cost Averaging)** — 추가 진입으로 평단을 조정합니다. 위험 관리에 양면성이 있습니다 — 잘 쓰면 강력하고, 잘못 쓰면 손실이 확대됩니다.

**DEX (Decentralized Exchange)** — Hyperliquid, dYdX, GMX, [Lighter](https://app.lighter.xyz/?referral=GMYPZWQK69X4) 등입니다. 주로 온체인입니다.

**Drawdown (낙폭)** — 자본의 고점 대비 손실입니다. Max Drawdown은 평가의 핵심 메트릭입니다.

## E

**EIP-712** — Ethereum 표준 typed-data 서명입니다. Hyperliquid, [Reya](https://app.reya.xyz/trade?referredBy=8src0ch8) 등이 사용합니다.

**Equity Tracker** — 거래소별 잔고를 주기적으로 기록합니다. 실제 PnL 측정의 진실의 원천입니다.

**Entry / Exit** — 진입 / 청산입니다. 시그널 디자인의 두 축입니다.

## F

**Factory Pattern** — 객체 생성 추상화입니다. 거래소 어댑터 등록 / 로드 패턴입니다.

**Funding Rate (펀딩비)** — Perp 거래소의 가격 / 인덱스 동조 메커니즘입니다. 보통 8시간마다 정산합니다.

**Funding Arbitrage (펀딩 차익)** — 거래소 간 / 자산별 펀딩비 차이를 차익으로 변환합니다.

## G

**Gas Fee (가스비)** — 온체인 트랜잭션 수수료입니다. EVM 체인은 ETH / BNB / MATIC 등입니다.

**Ghost Position** — DB에는 없지만 거래소에는 있는 포지션입니다. 큰 손실 원인이 됩니다.

**Graceful Restart** — 포지션 / 상태를 보존하며 재시작합니다. State manager가 필요합니다.

## H

**Hedging (헷지)** — 반대 포지션으로 노출을 줄입니다. 페어 트레이딩의 핵심입니다.

**HMAC** — Hash-based Message Authentication Code. 대부분 CEX의 서명 방식입니다.

**Hot Reload** — 봇 재시작 없이 config를 갱신합니다. SIGHUP 또는 파일 트리거를 사용합니다.

**Hyperliquid (HL)** — 메인 DEX입니다. 자체 L1, EIP-712 서명, Builder code 시스템을 사용합니다.

## I

**Iceberg Order** — 큰 주문을 작게 나눠 노출합니다. 잘 안 쓰지만 마켓메이커가 자주 사용합니다.

**IOC (Immediate-or-Cancel)** — 즉시 매칭이 안 되면 자동 취소합니다. 차익 봇에서 한쪽 노출 방지에 유용합니다.

**IP Whitelist** — API 키를 특정 IP에서만 사용 가능하게 합니다. 보안의 기본입니다.

## K

**Kelly Criterion** — 우위 기반 최적 베팅 사이즈 공식입니다. 풀 켈리는 너무 공격적이고, 1/4 켈리 정도가 실용적입니다.

**Kimchi Premium (김프)** — 한국 거래소 가격이 글로벌 거래소 대비 비쌉니다. 자본 흐름 비대칭 때문입니다.

## L

**Latency** — API 호출 / 응답 시간입니다. 차익 봇에는 critical합니다.

**Leverage (레버리지)** — 자본 대비 포지션 배수입니다. 페어 트레이딩 10x 정도가 기본입니다.

**Limit Order (지정가)** — 특정 가격에 매수 / 매도 주문입니다. Maker = 메이커 (오더북에 호가 추가) / Taker = 테이커 (호가 흡수).

**Liquidation (청산)** — 마진 부족 시 거래소가 강제로 포지션을 종료합니다. 레버리지 사용 시 항상 의식해야 합니다.

**Look-ahead Bias** — 백테스트에서 미래 정보가 누설된 채로 시뮬되는 오류입니다.

## M

**Maker / Taker** — 오더북에 유동성 제공 / 흡수입니다. 수수료가 다릅니다.

**Margin (마진)** — 포지션 보증금입니다. Leverage = position / margin.

**Mark Price** — 거래소가 평가 / 청산 기준으로 쓰는 가격입니다. 일반적으로 현물 인덱스 + 펀딩입니다.

**Market Order (마켓가)** — 즉시 매칭 가격에 매매합니다. 슬리피지가 발생합니다.

**MemKraft** — 사용하는 AI 메모리 시스템입니다. 옵시디언 통합 방식입니다.

**Momentum (모멘텀)** — 단기 가격 변화율입니다. 페어 트레이딩의 진입 시그널 중 하나입니다.

## N

**Notional Value** — 포지션의 명목 가치입니다. 사이즈 × 가격.

## O

**OHLCV** — Open / High / Low / Close / Volume. 캔들 데이터의 기본 단위입니다.

**Orphan Position** — 시스템 / 봇과 분리된 채 거래소에만 있는 포지션입니다. Ghost와 비슷합니다.

**Out-of-sample (OOS)** — 모델 훈련에 안 쓴 데이터로 평가합니다. Walk-forward에 사용합니다.

## P

**Paper Trade** — 실시간 가격 + 가짜 잔고로 시뮬합니다. 라이브 직전 검증 단계입니다.

**Pair Trading (페어 트레이딩)** — 두 상관 자산의 가격 비율 / 스프레드 평균회귀 베팅입니다.

**Perp (Perpetual Futures)** — 만기 없는 선물 계약입니다. Funding으로 인덱스와 동조합니다.

**PnL (Profit and Loss)** — 손익입니다. Realized (실현) / Unrealized (미실현).

**[Polymarket](https://polymarket.com/?ref=coinmage)** — Polygon 기반 예측시장입니다. 이진 outcome.

**Position** — 활성 포지션 (롱 / 숏 / 플랫).

**Post-Only** — 메이커로만 체결합니다. 테이커가 되면 자동 취소됩니다.

**Profit Factor** — Gross profit / Gross loss. > 1.5가 의미 있는 알파의 기준입니다.

## R

**R:R (Risk-Reward)** — 손절 폭 vs 익절 폭 비율입니다. 본전 승률을 결정합니다.

**Rate Limit** — API 호출 빈도 제한입니다. 거래소마다 다릅니다.

**Regime Filter (레짐 필터)** — 시장 상태 (상관관계 / 변동성 등) 기반 진입 조건입니다. 페어 트레이딩의 핵심 안전장치입니다.

**REST API** — HTTP 기반 API입니다. 폴링용입니다.

**Rolling Correlation** — 일정 윈도우의 상관계수입니다. 시간에 따른 변화를 추적합니다.

**RSI (Relative Strength Index)** — 모멘텀 오실레이터입니다. 70+ 과매수, 30- 과매도.

## S

**Sharpe Ratio** — 변동성 대비 수익률입니다. 1.5+가 안정적 알파입니다.

**Signal** — 진입 / 청산 결정의 기반입니다. z-score, momentum 등이 있습니다.

**Slippage (슬리피지)** — 주문 가격과 실제 체결 가격의 차이입니다.

**SDK (Software Development Kit)** — 거래소가 제공하는 클라이언트 라이브러리입니다.

**Spread (스프레드)** — bid-ask 차이 또는 두 자산 가격 차이입니다.

**State Manager** — 봇 상태 영속화입니다. graceful restart에 필수입니다.

**Strategy Evolver** — 시그널 가중치 자동 튜닝입니다. 6시간마다 성과 기반으로 조정합니다.

**StarkNet** — Layer 2입니다. [EdgeX](https://pro.edgex.exchange/referral/570254647), Paradex 등이 사용합니다.

## T

**Take Profit (TP) / Stop Loss (SL)** — 익절 / 손절가입니다. R:R 기준으로 디자인합니다.

**Telegram Bot** — 알림 / 원격 제어 인터페이스입니다. BotFather로 생성합니다.

**Tick** — 가격의 최소 단위입니다. 거래소 / 자산별로 다릅니다.

**Trailing Stop** — 가격이 좋아질 때마다 손절가가 따라 움직입니다. 수익 보호용입니다.

**Trigger File** — Windows 친화 봇 제어 패턴입니다. 파일 생성 → 봇이 감지 → 처리.

## V

**Venv (Virtual Environment)** — Python 의존성 격리입니다. 거래소 SDK 충돌 방지에 필수입니다.

**Volume Farming** — 거래량 기반 포인트 / 리워드 누적입니다. 양방향 헷지로 가격 노출 0.

## W

**Walk-forward** — 시간 순으로 train / test 반복입니다. 백테스트 견고성 검증에 사용합니다.

**WAL (Write-Ahead Log)** — SQLite의 동시성 모드입니다. 봇이 같은 DB에 동시 쓸 때 필요합니다.

**WebSocket (WS)** — 실시간 양방향 통신입니다. 시세 / 체결 알림에 사용합니다.

**Win Rate** — 승률입니다. 손익분기 승률 = SL / (TP + SL).

## Z

**z-score** — 평균에서 표준편차 단위로 얼마나 떨어졌는가를 나타냅니다. 페어 트레이딩의 핵심 시그널입니다.
