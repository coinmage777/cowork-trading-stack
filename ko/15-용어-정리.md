# 15. 용어 정리

가이드 전체에 나오는 용어를 알파벳 순으로. 처음 보는 사람을 위한 빠른 참조용.

## A

**Adverse Selection (역선택)** — 시장에서 주문이 체결되는 경우는 상대방이 정보 우위를 가진 경우. 봇 매매 시 체결 자체가 마이너스 시그널이 될 수 있음.

**Agent Wallet** — 메인 지갑이 트레이딩 권한을 위임한 별도 키. Hyperliquid 같은 DEX에서 메인 지갑 노출 없이 봇 운영 가능.

**Aiohttp** — Python 비동기 HTTP 라이브러리. 봇이 거래소 API에 비동기 호출할 때 표준.

**API Key / Secret** — 거래소 API 인증용. 트레이딩 권한만 주고 출금 권한은 절대 X.

**APR / APY** — Annual Percentage Rate / Yield. 펀딩 차익 등 전략의 연환산 수익률.

**Arbitrage (차익거래)** — 같은 자산이 다른 시장에서 다른 가격에 거래될 때, 양쪽 동시 진입으로 차익 실현.

**ATR (Average True Range)** — 변동성 지표. 동적 사이징 / 손절 폭 결정에 사용.

## B

**Backtest** — 과거 데이터에 전략을 시뮬해서 성과 측정. 라이브 박기 전 필수.

**Basis (베이시스)** — 현물 가격과 선물 / Perp 가격의 차이. 베이시스 차익은 carry trade의 한 형태.

**BBO (Best Bid/Offer)** — 오더북에서 매수 최고 / 매도 최저가. 지정가 진입 시 기준점.

**Bollinger Bands** — 가격의 표준편차 기반 밴드. 돌파 / 회귀 시그널 보조.

**Builder Code** — Hyperliquid의 라우팅 메커니즘. 프론트엔드(Miracle, DreamCash 등)별 코드로 수수료 분배.

## C

**ccxt** — 100+ 거래소 통합 Python / JS 라이브러리. CEX 봇의 표준.

**CEX (Centralized Exchange)** — 바이낸스, OKX, Bybit 등 중앙거래소.

**Circuit Breaker** — 일일 -X% 손실 시 자동 정지. 무한 손실 방지.

**Clearinghouse State** — Hyperliquid 등 DEX의 사용자 잔고 / 포지션 상태.

**CLOB (Central Limit Order Book)** — 중앙 오더북 매칭. Polymarket이 이 구조.

**Cloid** — Client Order ID. 거래소 주문 식별자. Miracle 같은 프론트엔드는 cloid prefix 필수.

**Correlation (상관관계)** — 두 자산 가격의 동조성. 페어 트레이딩의 레짐 필터에 사용.

**Cross-Venue** — 거래소 간. Cross-venue arb = 거래소 간 가격 차이 차익.

## D

**DCA (Dollar Cost Averaging)** — 추가 진입으로 평단 조정. 위험 관리에 양면성 — 잘 쓰면 강력, 잘못 쓰면 손실 확대.

**DEX (Decentralized Exchange)** — Hyperliquid, dYdX, GMX, Lighter 등. 주로 온체인.

**Drawdown (낙폭)** — 자본의 고점 대비 손실. Max Drawdown은 평가의 핵심 메트릭.

## E

**EIP-712** — Ethereum 표준 typed-data 서명. Hyperliquid, Reya 등이 사용.

**Equity Tracker** — 거래소별 잔고를 주기적으로 기록. 실제 PnL 측정의 진실의 원천.

**Entry / Exit** — 진입 / 청산. 시그널 디자인의 두 축.

## F

**Factory Pattern** — 객체 생성 추상화. 거래소 어댑터 등록 / 로드 패턴.

**Funding Rate (펀딩비)** — Perp 거래소의 가격 / 인덱스 동조 메커니즘. 보통 8시간마다 정산.

**Funding Arbitrage (펀딩 차익)** — 거래소 간 / 자산별 펀딩비 차이를 차익으로 변환.

## G

**Gas Fee (가스비)** — 온체인 트랜잭션 수수료. EVM 체인은 ETH / BNB / MATIC 등.

**Ghost Position** — DB에는 없지만 거래소에는 있는 포지션. 큰 손실 원인.

**Graceful Restart** — 포지션 / 상태 보존하며 재시작. State manager 필요.

## H

**Hedging (헷지)** — 반대 포지션으로 노출 줄이기. 페어 트레이딩의 핵심.

**HMAC** — Hash-based Message Authentication Code. 대부분 CEX의 서명 방식.

**Hot Reload** — 봇 재시작 없이 config 갱신. SIGHUP 또는 파일 트리거.

**Hyperliquid (HL)** — 메인 DEX. 자체 L1, EIP-712 서명, Builder code 시스템.

## I

**Iceberg Order** — 큰 주문을 작게 나눠 노출. 잘 안 쓰지만 마켓메이커가 자주.

**IOC (Immediate-or-Cancel)** — 즉시 매칭 안 되면 자동 취소. 차익 봇에서 한쪽 노출 방지에 유용.

**IP Whitelist** — API 키를 특정 IP에서만 사용 가능하게. 보안의 기본.

## K

**Kelly Criterion** — 우위 기반 최적 베팅 사이즈 공식. 풀 켈리는 너무 공격적, 1/4 켈리 정도가 실용적.

**Kimchi Premium (김프)** — 한국 거래소 가격이 글로벌 거래소 대비 비쌈. 자본 흐름 비대칭 때문.

## L

**Latency** — API 호출 / 응답 시간. 차익 봇에는 critical.

**Leverage (레버리지)** — 자본 대비 포지션 배수. 페어 트레이딩 10x 정도가 기본.

**Limit Order (지정가)** — 특정 가격에 매수 / 매도 주문. Maker = 메이커 (오더북에 호가 추가) / Taker = 테이커 (호가 흡수).

**Liquidation (청산)** — 마진 부족 시 거래소가 강제 포지션 종료. 레버리지 사용 시 항상 의식.

**Look-ahead Bias** — 백테스트에서 미래 정보가 누설된 채로 시뮬되는 오류.

## M

**Maker / Taker** — 오더북에 유동성 제공 / 흡수. 수수료 다름.

**Margin (마진)** — 포지션 보증금. Leverage = position / margin.

**Mark Price** — 거래소가 평가 / 청산 기준으로 쓰는 가격. 일반적으로 현물 인덱스 + 펀딩.

**Market Order (마켓가)** — 즉시 매칭 가격에 매매. 슬리피지 발생.

**MemKraft** — 쓰는 AI 메모리 시스템. 옵시디언 통합.

**Momentum (모멘텀)** — 단기 가격 변화율. 페어 트레이딩의 진입 시그널 중 하나.

## N

**Notional Value** — 포지션의 명목 가치. 사이즈 × 가격.

## O

**OHLCV** — Open / High / Low / Close / Volume. 캔들 데이터의 기본 단위.

**Orphan Position** — 시스템 / 봇과 분리된 채 거래소에만 있는 포지션. Ghost와 비슷.

**Out-of-sample (OOS)** — 모델 훈련에 안 쓴 데이터로 평가. Walk-forward에 사용.

## P

**Paper Trade** — 실시간 가격 + 가짜 잔고로 시뮬. 라이브 직전 검증 단계.

**Pair Trading (페어 트레이딩)** — 두 상관 자산의 가격 비율 / 스프레드 평균회귀 베팅.

**Perp (Perpetual Futures)** — 만기 없는 선물 계약. Funding으로 인덱스와 동조.

**PnL (Profit and Loss)** — 손익. Realized (실현) / Unrealized (미실현).

**Polymarket** — Polygon 기반 예측시장. 이진 outcome.

**Position** — 활성 포지션 (롱 / 숏 / 플랫).

**Post-Only** — 메이커로만 체결. 테이커가 되면 자동 취소.

**Profit Factor** — Gross profit / Gross loss. > 1.5 가 의미 있는 알파의 기준.

## R

**R:R (Risk-Reward)** — 손절 폭 vs 익절 폭 비율. 본전 승률 결정.

**Rate Limit** — API 호출 빈도 제한. 거래소마다 다름.

**Regime Filter (레짐 필터)** — 시장 상태 (상관관계 / 변동성 등) 기반 진입 조건. 페어 트레이딩의 핵심 안전장치.

**REST API** — HTTP 기반 API. 폴링용.

**Rolling Correlation** — 일정 윈도우의 상관계수. 시간 따라 변화 추적.

**RSI (Relative Strength Index)** — 모멘텀 오실레이터. 70+ 과매수, 30- 과매도.

## S

**Sharpe Ratio** — 변동성 대비 수익률. 1.5+ 가 안정적 알파.

**Signal** — 진입 / 청산 결정의 기반. z-score, momentum, 등.

**Slippage (슬리피지)** — 주문 가격과 실제 체결 가격 차이.

**SDK (Software Development Kit)** — 거래소가 제공하는 클라이언트 라이브러리.

**Spread (스프레드)** — bid-ask 차이 또는 두 자산 가격 차이.

**State Manager** — 봇 상태 영속화. graceful restart 필수.

**Strategy Evolver** — 시그널 가중치 자동 튜닝. 6시간마다 성과 기반 조정.

**StarkNet** — Layer 2. EdgeX, Paradex 등이 사용.

## T

**Take Profit (TP) / Stop Loss (SL)** — 익절 / 손절가. R:R 기준으로 디자인.

**Telegram Bot** — 알림 / 원격 제어 인터페이스. BotFather로 생성.

**Tick** — 가격의 최소 단위. 거래소 / 자산별 다름.

**Trailing Stop** — 가격이 좋아질 때마다 손절가가 따라 움직임. 수익 보호.

**Trigger File** — Windows 친화 봇 제어 패턴. 파일 생성 → 봇이 감지 → 처리.

## V

**Venv (Virtual Environment)** — Python 의존성 격리. 거래소 SDK 충돌 방지에 필수.

**Volume Farming** — 거래량 기반 포인트 / 리워드 누적. 양방향 헷지로 가격 노출 0.

## W

**Walk-forward** — 시간 순으로 train / test 반복. 백테스트 견고성 검증.

**WAL (Write-Ahead Log)** — SQLite의 동시성 모드. 봇이 같은 DB 동시 쓸 때 필요.

**WebSocket (WS)** — 실시간 양방향 통신. 시세 / 체결 알림에 사용.

**Win Rate** — 승률. 손익분기 승률 = SL / (TP + SL).

## Z

**z-score** — 평균에서 표준편차 단위로 얼마나 떨어졌는가. 페어 트레이딩의 핵심 시그널.

---

## 마치며

이 가이드는 매일 쓰는 시스템의 스냅샷이다. 시간이 지나면서 거래소가 사라지고, 새 거래소가 나타나고, 시그널이 안 통하게 되고, 새 시그널을 발견할 것이다. 그게 이 분야의 본질이다.

**이 가이드의 절반은 6개월 후 outdated.** 그래도 프레임워크 — 자동화 마인드셋, 데이터 기반 결정, 인프라 / 운영 원칙, AI 활용 — 는 변하지 않는다.

이 가이드로 누군가의 첫 자동화에 도움이 됐다면 그것으로 만족.

수익은 시스템에서 나온다. 시스템은 시간에서 나온다. 시간은 인내에서 나온다.

좋은 알파가 있길.
