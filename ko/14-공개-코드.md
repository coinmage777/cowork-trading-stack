## 추후 공개할 수도 있는 미니 도구

이 가이드를 적은 시점 기준으로, 다음 도구들은 정리만 잘 되면 공개해도 좋겠다고 생각하는 것:

### 1) PnL Reporter
거래소 잔고 스냅샷 → 일일 / 주간 PnL 자동 보고서. 매일 쓰는 스크립트의 일반화 버전.

### 2) Funding Rate Monitor
N개 거래소의 펀딩비 비교 → 차이 임계 도달 시 텔레그램 알림. 차익 봇 만들기 전 모니터링용.

### 3) Cross-Exchange Price Aggregator
여러 거래소의 mark price를 하나의 dashboard에. WebSocket 기반.

### 4) Backtest Skeleton
페어 트레이딩 백테스트 미니 프레임워크. config.yaml 입력 → 메트릭 출력. 8장의 학습용 정리본.

### 5) Bot Health Monitor
별도 프로세스로 봇 상태 모니터 → 좀비 / hung 감지 → 텔레그램 알림 + 자동 재시작.

이 도구들 공개 시 별도 repo 또는 이 repo의 `tools/` 디렉토리에 추가될 수 있음.

## 안 공개할 것 (왜)

### 1) 메인 페어 트레이딩 봇 풀 코드
이유:
- 알파 보호
- 환경 / 거래소 / 자본 규모에 맞춰져 있어 일반화 안 됨
- 카피해서 운영하면 사고 위험

### 2) 정확한 시그널 가중치 / 파라미터
이유:
- 시점별로 다름 (시장 변화 따라 조정)
- 데이터에 과적합된 값일 수 있음
- 다른 사람 환경에서 작동 보장 X

### 3) 운영해온 거래소 정확한 리스트 / 사이즈
이유:
- 셀프 트레이드 / 봇 식별 회피
- 자본 노출 회피

대신 이 가이드는 **프레임워크와 사고방식**을 전달한다. 그걸로 알파 만드는 게 맞는 길.

## 학습 순서 추천

1. **freqtrade** 로 시작해서 백테스트 / 페이퍼 / 라이브 흐름 익힘
2. **ccxt** 로 다양한 거래소 API 친숙
3. **단순 시그널 백테스트** 직접 코딩 (z-score / RSI / 모멘텀)
4. **Hyperliquid SDK** 로 첫 DEX 봇 (페이퍼)
5. **본격 봇 빌드** — 환경에 맞춰 from scratch

각 단계마다 코드 한 줄도 안 짤 자신 있는 사람은 잘못된 방향이다. AI가 90% 짜주는 시대지만, 그 코드를 이해 못하면 운영 사고 시 대응 못 함.
